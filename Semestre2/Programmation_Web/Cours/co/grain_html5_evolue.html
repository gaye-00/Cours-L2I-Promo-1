Fonctionnalités évoluées de HTML 5 : les formulaires

Les formulaires sont utilisés dans une page HTML pour permettre aux utilisateurs de fournir des informations comme des commentaires, un login et mot de passes, une adresse de livraison, un numéro de carte de crédit, le choix d'un produit, etc.

En d'autre terme, ce sont avec les liens hypertextuels, les éléments qui apportent de l'interactivité dans une page Web. Les balises qui sont manipulées dans une formulaires sont principalement : <form>, <input>, <legend>, <label>, <textarea>, <select>, <option>, <optgroup>, <button>, <progress>, <fieldset>

La balise <form> permet donc de regrouper l'ensemble des champs permettant de recueillir des informations auprès de l'utilisateur. La syntaxe générale de cette balise est présentée ci-après:

Dans l'utilisation des formulaires, on peut se poser deux questions :

  • Question 1: comment envoyer les informations fournies par l'utilisateur, Par quel moyen ?

  • Question 2: une fois que les informations ont été envoyées, comment les traiter ?

Les réponses à ces deux questions sont données par les attributs de la balise <form> :

  • method : cet attribut indique par quel moyen les données vont être envoyées (Réponse 1). Il existe deux solutions pour envoyer des données sur le Web :

    • method="get" : les informations (limitées à 255 caractères) seront envoyées dans l'adresse de la page (http://...).

    • method="post" : c'est la méthode la plus utilisée pour les formulaires car elle permet d'envoyer un grand nombre d'informations. Les données saisies dans le formulaire ne transitent pas par la barre d'adresse.

  • action : c'est l'adresse de la page ou du programme qui va traiter les informations (Réponse 2). Le traitement de ces informations ne peut pas se faire en HTML et CSS,. Il faudra utiliser d'autres langages comme PHP, JAVA, etc: C'est un des aspects dynamiques du web.
